01 Investor Brief

Executive summary for seed-stage investors

Problem

AI agents are entering production environments at an unprecedented rate — automating customer support, executing trades, managing infrastructure, and making autonomous decisions. Yet there is no runtime governance infrastructure for these systems. Organisations face a false binary: pause all AI deployment (losing competitive advantage) or ship ungoverned agents (accepting catastrophic risk). Both options are untenable.

Without an audit trail, policy enforcement, or anomaly detection layer, every AI agent in production today is effectively running blind. Regulators are beginning to notice — and the liability is mounting.

Solution — WitnessOS

WitnessOS is the first runtime governance layer purpose-built for autonomous AI agents. It sits between the agent and its execution environment, enforcing policies in under 50 milliseconds while maintaining a complete, tamper-evident cryptographic audit trail of every decision.

  • Policy Engine: Pre-flight validation of every agent action against configurable rules
  • Merkle-Chain Audit: WORM (Write-Once-Read-Many) storage with Ed25519 cryptographic signatures
  • Case Management: Phone-home system for human-in-the-loop escalation and review
  • Anomaly Detection: Real-time behavioural monitoring of agent actions

Product

  • 25-module FastAPI gateway engine
  • Merkle-chain WORM audit storage
  • Ed25519 cryptographic enforcement
  • Role-based access control (RBAC)
  • Phone-home case management
  • Real-time anomaly detection
  • Gmail dispatch integration
  • Stripe billing integration

Market Timing

The regulatory window is opening. The EU AI Act is now in force, Australia is building its AI governance framework, and every major jurisdiction is moving toward mandatory compliance requirements for autonomous systems.

"No dedicated competitor exists for AI agent runtime governance."

The AI agent market is projected to grow from $5.4B (2024) to $47.1B by 2030. Every one of those agents will need governance infrastructure.

How It Works

Agent → Policy Engine → Audit Chain → Case Mgmt
  1. Agent requests an action (API call, data access, system command)
  2. Policy Engine evaluates the action against configurable rules (<50ms)
  3. Approved actions are logged to the Merkle-chain audit trail with Ed25519 signatures
  4. Flagged actions trigger phone-home case creation for human review
  5. Full cryptographic audit trail is available for compliance, debugging, and reporting
Slide 5 of 11

Product Architecture

25-module FastAPI gateway engine with a modular, extensible architecture:

  • Core: FastAPI gateway, middleware chain, plugin system, configuration management
  • Audit: Merkle-chain WORM storage, Ed25519 signing, verification API, export tooling
  • Policy: Rule engine, RBAC system, policy compiler, dry-run mode
  • Case: Phone-home receiver, case management dashboard, Gmail dispatch, Slack notifications
  • Integrations: Stripe billing, Gmail dispatch, REST API, Python SDK
  • Agent: Agent #10 — built-in autonomous agent for internal governance tasks
